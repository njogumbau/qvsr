# Contributing to oTree-QVSR

We welcome contributions from the community! Whether you're reporting a bug, proposing a new feature, or improving the documentation, your help is appreciated.

## How to Contribute

### Reporting Bugs
If you find a bug, please [open an issue](https://github.com/njogumbau/qvsr/issues) on GitHub. Include:
- A clear, descriptive title.
- Steps to reproduce the bug.
- Expected vs. actual behavior.
- oTree version and browser details.

### Proposing New Features
We're always looking to expand the capabilities of oTree-QVSR. If you have an idea:
1. Check the existing issues to see if the feature has been proposed.
2. [Open a new issue](https://github.com/njogumbau/qvsr/issues) to discuss the feature before starting implementation.

### Submitting Pull Requests
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/your-feature-name`).
3. Implement your changes.
4. Ensure your code follows existing style conventions.
5. Add or update tests as appropriate.
6. Submit a pull request with a clear description of the changes.

## Development Setup

To develop oTree-QVSR, you'll need a working oTree environment. 
1. Install requirements: `pip install -r requirements.txt`.
2. Clone the repo into an oTree project folder.
3. Run the tests: `otree test qvsr`.
4. Run the server: `otree devserver`.

## Community Guidelines
Please be respectful and constructive in all interactions. We aim to maintain a welcoming environment for all contributors.

## Contact
For any other inquiries, please contact the repository maintainers.
